# Getting Started with FangOps

Welcome to **FangOps**! This guide will help you set up the open-source Agent Control Plane for IT and Business Operations locally.

## Prerequisite: API Keys

Before you start, you need at least one Large Language Model (LLM) API key.
- **OpenAI**: `sk-proj-...` (Default and recommended for `gpt-4o`)
- **Anthropic**: `sk-ant-...` (Supported for `claude-3-5-sonnet`)

You will also need **Docker** installed if you want to use the full `docker-compose` stack (which includes Prometheus and PostgreSQL).

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/fangops/fangops.git
cd fangops
```

### 2. Run the Setup Wizard

We've provided an interactive setup script to generate your `.env` file and configure your API keys.

```bash
# If using pnpm (recommended)
pnpm install
pnpm run setup

# Or using npm
npm install
npm run setup
```

Follow the prompts to enter your OpenAI key and (optionally) your Slack Webhook URL for the Reporter Hand.

### 3. Initialize the Database

FangOps uses SQLite by default for easy local setups.

```bash
# Push the Drizzle schema to the database
pnpm run db:push

# Seed the database with a default Admin user and some sample alerts
pnpm run db:seed
```

Your default login will be:
- **Email:** `demo@fangops.local`
- **Password:** `admin`

## Running the Application

There are two ways to run FangOps locally.

### Option A: Node.js Monorepo Dev Server (Recommended for Developers)

If you are developing new Hands or modifying the UI:

```bash
pnpm run dev
```

This will concurrently start:
- **API Server** at `http://localhost:3000`
- **React Dashboard** at `http://localhost:5173`
- The TypeScript build watchers for all core packages.

### Option B: Docker Compose (Recommended for Users)

If you want the full environment with Prometheus, Redis, and a production build of the Dashboard:

```bash
docker-compose up -d --build
```

The dashboard will be available at `http://localhost:3000` (Nginx proxies to the UI and API).

## Next Steps

Now that you are running:
1. Log into the Dashboard.
2. Navigate to the **Hands** page to view the active autonomous agents.
3. Check the **Incidents** page to see the sample data generated by the seed script.
4. Read the [Hands Architecture](./hands.md) documentation to learn how to add your own runbooks and custom adapters.
